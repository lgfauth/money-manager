## Directory Guide

1. [architecture](architecture): architecture and system boundaries.
2. [guides](guides): active development guides and standards.
3. [operations](operations): deployment and runtime operations.
4. [troubleshooting](troubleshooting): recurring failures and diagnostics.
5. [history](history): selected historical notes with ongoing maintenance value.
6. [archive](archive): superseded docs kept for traceability only.

## Documentation Rules

1. New implementation rules go to [guides](guides), not top-level ad-hoc files.
2. Architecture changes must update [architecture/architecture-overview.md](architecture/architecture-overview.md).
3. Documents that are completed, duplicated, or obsolete should be moved to [archive](archive).
4. Historical files in [history](history) should be kept only if they still help maintenance today.
